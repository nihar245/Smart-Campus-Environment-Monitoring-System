# AWS Setup Guide - Smart Campus Environment Monitoring System

This guide explains everything to do in AWS from scratch, in the exact order, so your React frontend can consume live data generated by Lambda and stored in DynamoDB.

## 0. What You Are Building

You will deploy:

1. DynamoDB tables for room config and sensor readings
2. Lambda functions for:
- generating random sensor values
- serving API endpoints
3. API Gateway REST API with 3 GET routes
4. EventBridge schedule to run generator periodically
5. S3 static website hosting for frontend

---

## 1. AWS Account and Region Basics

1. Sign in to AWS Console.
2. Choose one region and use it everywhere (example: ap-south-1).
3. Open IAM and create an admin user (if not already done).
4. Enable billing alerts (recommended).

Notes:
- Keep all resources in the same region.
- Save your region choice and use it consistently.

---

## 2. Create DynamoDB Tables

Open DynamoDB and create these 3 tables.

## 2.1 Table A: SmartCampusReadings

Purpose: full historical data

1. Table name: SmartCampusReadings
2. Partition key: roomId (String)
3. Sort key: timestamp (String)
4. Capacity mode: On-demand

## 2.2 Table B: SmartCampusLatest

Purpose: only latest row per room (fast dashboard load)

1. Table name: SmartCampusLatest
2. Partition key: roomId (String)
3. Capacity mode: On-demand

## 2.3 Table C: SmartCampusRooms

Purpose: static room metadata

1. Table name: SmartCampusRooms
2. Partition key: roomId (String)
3. Capacity mode: On-demand

---

## 3. Create IAM Role for Lambda

1. Open IAM > Roles > Create role.
2. Trusted entity: AWS service
3. Use case: Lambda
4. Role name: SmartCampusLambdaRole
5. Attach managed policy: AWSLambdaBasicExecutionRole

Add inline DynamoDB policy to this role:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "dynamodb:PutItem",
        "dynamodb:BatchWriteItem",
        "dynamodb:UpdateItem",
        "dynamodb:GetItem",
        "dynamodb:Query",
        "dynamodb:Scan"
      ],
      "Resource": "*"
    }
  ]
}
```

For production, replace `"*"` with exact table ARNs.

---

## 4. Deploy Sensor Generator Lambda

You already have generator code in backend/lambda/function.py.

1. Open Lambda > Create function.
2. Author from scratch.
3. Function name: sensor-generator
4. Runtime: Python 3.11
5. Execution role: Use existing role > SmartCampusLambdaRole
6. Create function.
7. Replace default code with function.py code.
8. Deploy.

Set environment variables (Configuration > Environment variables):

1. TABLE_NAME = SmartCampusReadings

If you extend code to write latest snapshot too, add:

1. LATEST_TABLE_NAME = SmartCampusLatest

Test:

1. Click Test.
2. Create test event with empty JSON `{}`.
3. Run test.
4. Check CloudWatch logs for success.
5. Verify data appears in DynamoDB.

---

## 5. Seed SmartCampusRooms Table (One-Time)

You must populate room metadata once.

Each item should include:

1. roomId
2. floor
3. roomType
4. capacity
5. parentRoom
6. displayName

Options:

1. Use DynamoDB console manual import
2. Use a one-time Lambda script
3. Use AWS CLI `batch-write-item`

Minimum requirement: all 53 rooms exist in SmartCampusRooms.

---

## 6. Create API Lambda Functions

Create 3 additional Lambda functions (Python 3.11, same role).

## 6.1 get-latest-readings

Purpose:

1. Scan SmartCampusLatest
2. Return JSON array of latest records for all rooms

Environment variables:

1. LATEST_TABLE_NAME = SmartCampusLatest

## 6.2 get-room-history

Purpose:

1. Read roomId from path parameter
2. Read minutes from query string (default 30)
3. Query SmartCampusReadings by roomId
4. Filter by timestamp window
5. Return oldest to newest

Environment variables:

1. READINGS_TABLE_NAME = SmartCampusReadings

## 6.3 get-rooms

Purpose:

1. Scan SmartCampusRooms
2. Return all room metadata

Environment variables:

1. ROOMS_TABLE_NAME = SmartCampusRooms

All API Lambdas should return CORS headers:

1. Access-Control-Allow-Origin: *
2. Access-Control-Allow-Headers: *
3. Access-Control-Allow-Methods: GET,OPTIONS
3. Content-Type: application/json

---

## 7. Create API Gateway REST API

1. Open API Gateway > Create API > REST API.
2. API name: smart-campus-api
3. Create resources and methods:

1. GET /readings/latest -> get-latest-readings Lambda
2. GET /readings/room/{roomId} -> get-room-history Lambda
3. GET /rooms -> get-rooms Lambda

Enable CORS for all resources.

Deploy:

1. Actions > Deploy API
2. Stage name: prod
3. Copy invoke URL (example: https://abc123.execute-api.ap-south-1.amazonaws.com/prod)

---

## 8. Create EventBridge Scheduler

Generator must run automatically.

1. Open EventBridge > Scheduler > Create schedule.
2. Name: smart-campus-generator-schedule
3. Schedule type: Recurring
4. Expression: rate(1 minute)
5. Target: Lambda function > sensor-generator
6. Execution role: allow scheduler to invoke Lambda
7. Create schedule.

Important:

1. Native EventBridge minimum interval is 1 minute.
2. If you truly need 30-second generation, use a different orchestration approach.

---

## 9. Configure Frontend Environment

In your project root, create `.env`:

```env
VITE_API_URL=https://<api-id>.execute-api.<region>.amazonaws.com/prod
```

Then run:

```bash
npm run build
```

---

## 10. Deploy Frontend to S3

1. Open S3 > Create bucket.
2. Bucket name must be globally unique.
3. Uncheck "Block all public access" (if using public static hosting).
4. Enable Static website hosting.
5. Index document: index.html
6. Error document: index.html
7. Upload all files inside local `dist/` folder.

Bucket policy for public read (replace bucket name):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
    }
  ]
}
```

Open the S3 website endpoint and verify app loads.

## 12. Monitoring and Troubleshooting

Use CloudWatch Logs for all Lambdas:

1. sensor-generator logs successful writes
2. API Lambdas show request/response errors

Common issues:

1. CORS error:
- Ensure API Gateway CORS enabled for all routes
- Ensure Lambda returns Access-Control-Allow-Origin

2. Empty frontend data:
- Verify `.env` URL is correct
- Verify API stage is `prod`
- Verify tables have data

3. Lambda access denied:
- Check SmartCampusLambdaRole permissions

4. History endpoint slow:
- Ensure query uses roomId partition key on SmartCampusReadings

---

## 13. Security and Production Hardening (After MVP)

1. Restrict IAM policies to exact table ARNs
2. Put API Gateway behind custom domain + WAF
3. Use CloudFront in front of S3
4. Add throttling and API keys if needed
5. Add DLQ/retry strategy for generator

---

## 14. Cost Awareness

For student-scale workload, cost is usually low with:

1. DynamoDB on-demand
2. low-frequency Lambda execution
3. API Gateway + S3 static hosting

Still, set AWS Budgets alerts to avoid surprises.
